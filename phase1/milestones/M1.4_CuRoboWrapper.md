# M1.4 cuRobo 封装 — IK + 整臂碰撞 + 轨迹代价

## 目标

> 把 cuRobo 的 IK / FK / 碰撞检测包装成 Phase 1 用得顺手的接口。屏蔽 cuRobo 的 batch / cuda graph / config 复杂度。

## 在管线中的位置

```text
M1.7 单 cycle:
  for each candidate ee_pose:
    theta = arm_kin.ik(ee_pose)        ← 这里
    if theta is None: skip
    if arm_kin.collides(theta, vm): skip
    
M1.5 全局图加边时:
    if arm_kin.edge_free(theta_a, theta_b, vm): add edge
    cost = arm_kin.trajectory_cost(theta_a, theta_b)
```

## 关键设计决定

| 决定 | 选项 | 选 | 理由 |
|------|------|------|------|
| Voxel→cuRobo 集成 | 体素 cube list vs custom collision | cube list | 简单：每个 occupied 体素 export 成一个 cuboid；性能够用 |
| IK 多解 | 单解 vs batch 多 seed | batch (20 seeds) | cuRobo 默认就是 batch，免费的多解 |
| Edge 碰撞 | 直线插值 K 步 vs trajopt | 直线插值 | Phase 1 简单；trajopt 在 Phase 2 才用 |
| 距离度量 | 关节空间 L2 vs cartesian | 关节空间 | 真实运动代价；cartesian 假象不准 |
| Robot frame | base frame vs world frame | 内部 base，输入输出 world | 用户面对 world，cuRobo 内部用 base |

import torch
from typing import Optional


class ArmKin:

        ...

        """前向运动学.
        theta: (N, 6) 关节角.
        return: {

        self,

        """逆运动学. 输入 world 系目标位姿.
        return: (N, 6) 关节角, 或 None 若全部失败.

        """
        ...

        theta: torch.Tensor,             # (N, 6)

        """整臂碰撞. 返回 (N,) bool, True = 碰撞."""

        voxel_map: "VoxelMap",

    ) -> bool:
        """两构型间直线插值, 每 step_rad 取一个中间构型, 全部不撞才返回 True."""
        ...

        self,
        theta_a: torch.Tensor,           # (N, 6)
        theta_b: torch.Tensor,           # (N, 6)

        """关节空间 L2 距离. 返回 (N,)."""
        ...

    def world_to_curobo_obstacles(

        voxel_map: "VoxelMap",

        """

## 实施步骤

### Step 1: 加载机器人配置

```python
def __init__(self, robot_yml_path, robot_pose_world, device):
    from curobo.types.base import TensorDeviceType
    from curobo.types.robot import RobotConfig
    from curobo.util_file import load_yaml
    from curobo.cuda_robot_model.cuda_robot_model import CudaRobotModel
    from curobo.wrap.reacher.ik_solver import IKSolver, IKSolverConfig
    
    self.tensor_args = TensorDeviceType()
    yml = load_yaml(robot_yml_path)
    self.robot_cfg = RobotConfig.from_dict(yml['robot_cfg'], self.tensor_args)
    self.kin = CudaRobotModel(self.robot_cfg.kinematics)
    
    self.robot_pose_world = robot_pose_world if robot_pose_world is not None else np.eye(4)
    
    # IKSolver 默认无 world (M1.4 单独做碰撞)
    ik_cfg = IKSolverConfig.load_from_robot_config(
        self.robot_cfg, None,
        rotation_threshold=0.05, position_threshold=0.005,
        num_seeds=20,
        self_collision_check=True,
        self_collision_opt=True,
        tensor_args=self.tensor_args,
        use_cuda_graph=True,
    )
    self.ik_solver = IKSolver(ik_cfg)
```

### Step 2: FK（含 world 系输出）

```python
def fk(self, theta):
    state = self.kin.get_state(theta)
    ee_pos_base = state.ee_position    # (N, 3)
    ee_quat_base = state.ee_quaternion # (N, 4)  [w,x,y,z]
    
    # base → world
    R_wb = torch.tensor(self.robot_pose_world[:3, :3], device=self.device, dtype=torch.float32)
    t_wb = torch.tensor(self.robot_pose_world[:3, 3],  device=self.device, dtype=torch.float32)
    ee_pos_world = (R_wb @ ee_pos_base.T).T + t_wb
    # 四元数变换: q_world = q_wb * q_base
    q_wb = matrix_to_quat_wxyz(R_wb)
    ee_quat_world = quat_mul(q_wb, ee_quat_base)
    
    return {
        'ee_pos_base': ee_pos_base,
        'ee_quat_base': ee_quat_base,
        'ee_pos_world': ee_pos_world,
        'ee_quat_world': ee_quat_world,
    }
```

### Step 3: IK（world → base 转换 + 调 cuRobo）

```python
def ik(self, ee_pos_world, ee_quat_world, n_seeds=20):
    from curobo.types.math import Pose
    
    # world → base
    R_bw = self.robot_pose_world[:3, :3].T  # base←world
    t_bw = -R_bw @ self.robot_pose_world[:3, 3]
    ee_pos_base = R_bw @ ee_pos_world + t_bw
    # quat: q_base = inv(q_wb) * q_world
    q_wb = ...
    q_base = quat_mul(quat_conj(q_wb), ee_quat_world)
    
    pos_t = torch.tensor(ee_pos_base.reshape(-1, 3), device=self.device, dtype=torch.float32)
    quat_t = torch.tensor(q_base.reshape(-1, 4), device=self.device, dtype=torch.float32)
    goal = Pose(pos_t, quat_t)
    
    result = self.ik_solver.solve_batch(goal)
    if result.success.any():
        return result.solution  # (N, 1, 6) 取第 0 个: [..., 0, :]
    return None
```

### Step 4: 碰撞（用 RobotWorld）

```python
def collides(self, theta, voxel_map):
    # 把 voxel_map 转成 cuRobo WorldConfig
    world_cfg = self.world_to_curobo_obstacles(voxel_map)
    # 创建 RobotWorld (可缓存, 但 voxel 每次变要重建)
    from curobo.wrap.model.robot_world import RobotWorld, RobotWorldConfig
    cfg = RobotWorldConfig.load_from_config(self.robot_cfg, world_cfg, ...)
    rw = RobotWorld(cfg)
    
    # 关键: voxel 系坐标和 base 系坐标的转换在 world_to_curobo_obstacles 里完成
    # theta 在 base frame, world_cfg 也在 base frame
    
    d_world, d_self = rw.get_world_self_collision_distance_from_joints(theta.unsqueeze(1))
    # < 0 = 碰撞
    return (d_world < 0).any(dim=-1) | (d_self < 0).any(dim=-1)
```

### Step 5: voxel → WorldConfig

策略：把 occupied 体素聚合成 axis-aligned cuboid。简单实现：每个 occupied 体素一个 cuboid。

```python
def world_to_curobo_obstacles(self, voxel_map):
    from curobo.geom.types import Cuboid, WorldConfig
    
    # 取所有 occupied 体素的中心 (在 world frame)
    occ_idx = (voxel_map.state == 2).nonzero()  # (M, 3)
    if len(occ_idx) == 0:
        return WorldConfig(cuboid=[])
    centers_world = voxel_map.index_to_world(occ_idx)  # (M, 3)
    
    # world → base
    R_bw = self.robot_pose_world[:3, :3].T
    t_bw = -R_bw @ self.robot_pose_world[:3, 3]
    centers_base = (R_bw @ centers_world.cpu().numpy().T).T + t_bw
    
    # 每个体素: cuboid dims = (res, res, res), pose 在中心
    cubes = []
    for i, c in enumerate(centers_base):
        cubes.append(Cuboid(
            name=f"vox_{i}",
            pose=[*c.tolist(), 1, 0, 0, 0],
            dims=[voxel_map.res] * 3,
        ))
    return WorldConfig(cuboid=cubes)
```

**警告**：M = 几千个 cuboid 时 cuRobo 会变慢。优化方向：

- 把相邻 occupied 体素合并成大 cuboid（greedy box packing）
- 只取距 robot reach 1 m 内的体素
- 或者把 voxel grid 直接做成 ESDF（占用 cuRobo 自己的 voxel collision API，本期略）

Phase 1 先不优化，<200 cuboid 应该够。

    """FK 算 EE pose, IK 反推, 关节角应该恢复 (允许 IK 多解, 但 EE pose 一致)."""

## 注意事项

1. **第一次 IK 调用慢（建 CUDA graph）**：测试时先调一次预热再计时。
2. **base ↔ world 变换易错**：建议写一个 `pose7_to_mat4` 工具函数集中处理。
3. **体素 → cuboid 转换可能慢**：M1.7 单 cycle 调用前需缓存（每个 cycle 一次）。
