```python
# phase1/depth_source.py 增加

class IsaacSimDepth:
    """实现 DepthSource 接口, 用 IsaacSimEnv 拍真渲染深度."""

    def __init__(self, env: IsaacSimEnv, theta_solver: callable):
        """env: 已启动的 IsaacSimEnv 实例.
        theta_solver: callable(camera_pose_world) -> joint angles.
                      用 ArmKin.ik 实现.
        """
        self.env = env
        self.theta_solver = theta_solver

    def render(self, camera_pose: np.ndarray) -> torch.Tensor:
        """先 IK 算关节角, 设给 sim, 再渲染."""
        theta = self.theta_solver(camera_pose)
        if theta is None:
            raise ValueError("Camera pose not reachable, can't render in IsaacSimEnv")
        self.env.set_joint_angles(theta)
        self.env.step()                # 让相机更新位姿
        return self.env.render_depth()
```

### Step 2: IsaacSimDepth 包装

详见接口规范。关键是 `render(camera_pose)` 内部要做 IK 反算关节角。

### Step 3: 处理 IK 失败

如果某个相机位姿对应的 IK 找不到合法关节角，`IsaacSimDepth.render()` 应该抛异常，让上层（M1.7 / M1.8）跳过这个候选。

实际上这种情况在 Phase 1 的主流程里不应发生——M1.7 已经过滤了 IK 失败。但 M1.8 的 global step 直接执行 graph 路径时，path 上每个节点的 theta 都已知，**不需要再算 IK**。所以 IsaacSimDepth.render 可以扩展接受 (camera_pose, theta)：

```python
def render(self, camera_pose, theta=None):
    if theta is None:
        theta = self.theta_solver(camera_pose)
    self.env.set_joint_angles(theta)
    self.env.step()
    return self.env.render_depth()
```

```python
    env.step()
    
    cam_pose = env.get_camera_pose()
    print("camera pose:", cam_pose)
    
    depth = env.render_depth()
    print("depth shape:", depth.shape, "mean:", depth.mean().item())
    
    assert depth.shape == (360, 640)
    assert depth.mean().item() > 0          # 至少看到了点东西
    
    env.shutdown()


def test_raycast_vs_isaacsim_consistency():
    """同一个相机位姿, raycast 和 isaacsim 渲染的深度应大致一致."""
    env = IsaacSimEnv(...)
    pkl_theta = pkl['joint_angles']
    env.set_joint_angles(pkl_theta)
    env.step()
    cam_pose = env.get_camera_pose()
    isaac_depth = env.render_depth().cpu().numpy()
    
    # raycast
    rsim = RaycastSim(mesh_paths=[BEAM_OBJ])
    rcast_depth = rsim.render(cam_pose).cpu().numpy()
    
    # 比较: 平均误差应该小, 但因为采样、抗锯齿等因素不会完全一致
    valid = (isaac_depth > 0) & (rcast_depth > 0)
    diff = np.abs(isaac_depth[valid] - rcast_depth[valid])
    print(f"mean abs diff: {diff.mean():.4f} m, max: {diff.max():.4f} m")
    assert diff.mean() < 0.05      # 平均 < 5 cm
```

## 注意事项

1. **AppLauncher 必须在 import omni 之前**（之前 visualize_seam_scene.py 有相同坑）。

3. **相机 prim 的 USD 位姿**需要和 cuRobo FK 输出一致——可能需要调一个 "camera_offset_in_ee_link" 参数对齐。
4. **物理 step**：当前不 step 物理（机械臂用 kinematic 模式）。如果要做动力学验证，再开。
