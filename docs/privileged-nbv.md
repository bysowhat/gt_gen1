# 特权 NBV（Privileged Next-Best-View）详述 

本文详述 GT 生成流水线中**最核心的一环**：如何用真值场景为机械臂选择
「下一最优视点」，从而高效、合规地揭开通往目标的走廊。

> 关联文档：
> - [`gt-generation-design.md`](./gt-generation-design.md) §4（特权专家思想）、§5（主循环）
> - [`gt-generation-curobo-implementation.md`](./gt-generation-curobo-implementation.md) §5.2、§6（主循环）

---

## 1. 定位：NBV 与「特权」各是什么

**NBV（Next-Best-View，下一最优视点）** 是主动感知的经典问题：

> 给定当前部分地图，**下一步把传感器移到哪，能获得最有用的新信息？**
> 评估一批候选视点 → 给每个打「信息增益」分 → 选最高者移过去 → 重复。

**普通 NBV 的痛点**：它不知道未知区里有什么，只能**估计**信息增益（数视锥内未知
体素、离前沿多近等启发式），估不准 → 探索低效 → 轨迹脏。

**「特权（privileged）」的含义**：我们在仿真里产 GT，**手握完整真值场景**，于是
可以**不估计、直接精确计算**某视点能揭开哪些未知体素（对真值 raycast 即可）。

这是模仿学习里的 **teacher-student / privileged expert** 范式：**老师（GT 生成器）
允许用真值「作弊」产出最优行为，学生模型只用真实观测去模仿。**

---

## 2. 关键前提：目标导向 NBV，不是通用探索 NBV

这一点最易做错。通用 NBV 要把**整个空间探索完**（最大化总覆盖）。但本任务**不是
探索整个房间**，而是：

> **只揭开「足够让手臂保守地走到目标」的那条走廊，多一点都不必要。**

因此打分函数**绝不能**是「数所有揭开的未知体素」——那会奖励去看与目标无关的大片
空地，产出绕远的脏 GT。**信息增益必须偏向「揭开通往目标的走廊」。**

---

## 3. 核心算法：全知路径引导（Oracle-Path-Guided NBV）

最优雅、最贴合需求的做法。核心洞察是一个反直觉但关键的事实：

> **在真值地图上，从当前位姿到目标一定能算出一条真正无碰撞的「全知最优路径」P\*。**
> **你现在走不了它，唯一原因不是有障碍——是路上有些地方还没被「确认为空」。**
> （P\* 在真值上无碰撞，所以挡你的**只可能是 UNKNOWN，绝不会是真障碍**。）

于是「往哪看」的答案变得清晰：**沿你打算走的理想路径往前看，确认它是空的，再走过去。**

### 算法四步

```
第1步：真值上算全知最优路径 P*（当前位姿 → 目标）
第2步：沿 P* 找"阻塞段 B" = reach_pt 之后一小段扫掠体积里【当前仍 UNKNOWN】的体素
       （这些就是"挡着你走理想路"的、待确认的格子）
第3步：生成一批"在已确认自由区内可达、且相机能看向 B"的候选视点
第4步：用真值精确算每个候选能揭开 B 中多少格子，选最多的那个
```

### 图示

```
   当前位姿 ●━━━━━━━━┓ P*(真值最优路,真的没障碍)
              已确认  ┃
              自由区   ┃░░░░░  ← 这段扫掠体积还是 UNKNOWN = 阻塞段 B
                       ┃░░░░░
                       ┗━━━━━━● 目标
        NBV = 选一个视点，把相机对准 B 这片，确认它是空的
        → 确认后 B 变 FREE → 走廊打通 → 保守规划就能往前走
```

**效果**：手臂表现得像一个聪明的人——朝着要去的方向看一眼、确认路通、走过去；
遇障碍时 P\* 本身已绕开，于是它会**朝绕行方向看**。产出的 GT 自然、高效、几乎
没有浪费动作。

## 4. 单轮 NBV 与完整主循环（含第一步到第七步详细步骤）

> ⚠️ 易混淆点：**§3 的四步只是「一次迭代」**，它只挑出**下一个**视点。从初始
> 到目标的完整过程，是主循环把这套迭代**重复很多轮**——目标越远，轮数越多。

### 4.1 移动到 NBV 视点 = 机械臂在「前进」

NBV 选出的视点是**沿 P\* 往前、且当前自由区内能到达**的构型。所以：

> **「移动到 NBV 视点」这个动作本身，就让机械臂沿 P\* 朝目标推进了一段，同时
> 相机顺便看清了前方下一段。前进与探看是同一个动作。**

目标很远 = 这个「探看+前进」动作重复 N 次，机械臂像挤牙膏一样一段段往前爬。

### 4.2 B 是一个「滑动窗口」

B 只取 reach_pt 之后**最近的一小段**未知（不是整条剩余路）。每轮 reach_pt 朝目标
爬一段，B 这个小窗口也跟着往前挪——**每轮只确认下一段**，这正是远目标逐段推进
的机制。

### 4.3 「规划到目标」是终点判定，⑤才是前进引擎

主循环每轮都试「在确认自由区内直接规划到目标」：
- 早期必然失败（剩余走廊没确认完）——**正常，不是错误**；
- 随着一轮轮前进，剩余未确认走廊越来越短，**到某轮全部确认，规划成功 → 收尾**。

真正驱动机械臂前进的，是每轮「移动到 NBV 视点」这一步。

### 4.4 具体例子（目标在大障碍后面、离起点很远）

| 轮次 | 机械臂在哪 | 这一轮发生什么 |
|---|---|---|
| 起点 | retract | 相机看到周围 ~30cm，算出 P\*（绕过大障碍通向目标）；确认自由区只是起点小气泡 |
| 第1轮 | retract | 规划到目标→失败（20cm 外被未知挡住）。选 P\* 前方一个可达视点→前移 ~15cm 并看清下一段 |
| 第2轮 | 沿 P\* 前进 15cm | 仍失败。再选视点、再前进、再揭开（开始看到大障碍近面） |
| 第3~5轮 | 绕着大障碍逐段爬 | 每轮「前进一段 + 看清下一段」，走廊一节节贴着障碍绕过去 |
| 第6轮 | 已绕过障碍 | 规划到目标→**成功**（剩余走廊已全部确认）→执行最后一段直奔目标 |
| 结束 | 目标位姿 | GT = 把所有这些移动段首尾拼起来的完整轨迹 |

**整条 GT 就是这一系列「探看+前进」段拼成的**；目标越远，中间轮次越多。

---

### 4.5 完整主循环逐步详解（准备阶段 + ①~⑦）

每一步「做什么 / 输入 / 调用 / 关键澄清 / 坑」讲透；NBV（§3 四步）嵌在 ②③④。

#### 准备阶段（进入循环前，含「冷启动」）

破解眼在手死锁的起点：

- **初始化三态图**：ROI 范围内所有体素设 `UNKNOWN`。
- **冷启动：机械臂当前本体体积 = FREE**。关键 bootstrapping——机械臂此刻物理上
  就停在 retract_config 且没碰撞，所以它本体占据的体素、以及周围相机当前能看到
  的，必然是自由的。先做一次观测，把起点周围一小块从 UNKNOWN 变 FREE，这样才有
  「第一块立足之地」。
- **求目标关节角**：用 cuRobo IK 把 `goal_pose` 解成 `goal_config`（碰撞感知）。
- **算全知最优路径 P\***：真值场景上规划 `config → goal_config`，得到 P\*。
- `config = retract_config`，`GT = [config]`。

---

#### ① 试着在【已确认自由区】内直接规划到目标

**做什么**：用 cuRobo 在「当前保守碰撞世界」里规划一条 `config → goal_pose` 的完整
无碰撞轨迹。

- **输入**：当前 `config`、`goal_pose`、当前 voxmap（非 FREE 全部当障碍）。
- **先做**：`sync_collision_world(voxmap)`——把「OCCUPIED ∪ UNKNOWN」全部灌进
  cuRobo 的 voxel 碰撞世界。
- **调用**：`motion_gen.plan_single(config, goal_pose, ...)`。
- **成功的含义**：存在一条**整条都落在已确认自由区**的可执行轨迹 → 接到 GT，结束。
- **关键澄清**：这条最终执行的路**由 cuRobo 自己算，不一定等于 P\***。P\* 只是
  「特权向导」，指导往哪看；真正落进 GT 的路是 cuRobo 在确认自由区里的最优解。
- **为什么每轮都试**：这步很便宜；走廊一旦确认完就能立刻收尾，不必多探。
- **坑**：早期几乎必然失败（走廊没确认完），这是**正常的，不是错误**。

---

#### ② 找阻塞段 B（reach_pt 之后那一小段未知）

**做什么**：算出沿 P\* 目前最远能保守走到哪（reach_pt），及正挡在前面那一小段未知 B。

##### (1) 先搞清楚 P\* 是什么数据

P\* 不是抽象的线，是 cuRobo 输出的**一串密集的关节构型**：

```
P* = [ q0, q1, q2, q3, ......, qN ]
       ↑                        ↑
    当前构型(起点)            目标构型
```

- 每个 `qi` 是一个**完整关节角向量**（如 6 轴臂 `(θ1..θ6)`）。
- 这些点是沿轨迹**密集插值**出来的，相邻两点只差一点点。
- **「逐点」就是指这一串 `q0, q1, q2, …`**。

##### (2) 求 reach_pt：沿 P\* 逐点向前扫

**「向前扫」**：从 q0（当前位置）开始，**按下标递增方向**一个个检查到 qN（目标）。

每个点检查什么 —— 关键：**P\* 上的点是「关节角」，但检查发生在「3D 工作空间」**：

- 机械臂摆成构型 `qi` 时，**整条手臂本体**占据 3D 空间一团体积。
- **「扫掠体积」**：从 `qi` 移到 `qi+1` 这一小段，手臂本体扫过的那团 3D 体积
  （用 cuRobo 碰撞球近似）。
- **「⊆ FREE」**：这团体积碰到的**每一个体素都必须是已确认 FREE**；只要碰到一个
  UNKNOWN 或 OCCUPIED 就不满足。

> 直白说：检查「这个构型/这一小段移动里，整条胳膊有没有任何部分伸进了『没确认
> 为空』的格子」。

**停止规则**：一路向前扫，**第一次发现扫掠体积碰到非 FREE 体素的那个点**就停，
**它前一个点 = reach_pt**。

```
沿 P* 向前扫：
  q0 ✅整臂都在FREE   q1 ✅   q2 ✅ ...... q37 ✅   q38 ❌(手臂碰到UNKNOWN)
                                            ↑           ↑
                                        reach_pt    第一个出界的点
  → 能保守地沿 P* 一路走到 q37，再往前(q38)就踏进未知了
```

```python
reach_idx = 0
for i in range(len(P_star) - 1):
    swept = swept_volume(P_star[i], P_star[i+1], robot_model)  # 这一小段整臂扫过的体素
    if all(voxmap[v] == FREE for v in swept):                  # 全部已确认空?
        reach_idx = i + 1                                       # 这段能走,前进
    else:
        break                                                  # 撞到非FREE,停
reach_pt = P_star[reach_idx]
```

##### (3) 求 B：reach_pt 前方一小段里仍 UNKNOWN 的体素

**B 就是**：从 reach_pt 再往前「一小段」（接下来 k 个路点，如 `q38~q43`），手臂在
这段里扫过的所有体素中，**目前还是 UNKNOWN 的那些**。

```
沿 P*：  ...─q37 ──── q38 q39 q40 q41 q42 q43 ──── ... ─ qN(目标)
              ↑       └─────── 往前一小段(k个点) ──────┘
          reach_pt    这段手臂扫过的体素里, 仍 UNKNOWN 的 ===> B
```

**为什么只取 UNKNOWN**（这段扫过的体素按三态分三类）：

| 体素状态 | 是否进 B | 原因 |
|---|---|---|
| **FREE** | ❌ | 已确认空，不挡路，不用再看 |
| **OCCUPIED** | ❌ | 已知障碍，本就该绕，不是「待确认」 |
| **UNKNOWN** | ✅ **就是 B** | **正因没确认才不敢走**——看清就能前进 |

> 一句话：**B = 「卡住你下一步的、唯一原因是『还没看过』的那些格子」**。

**为什么只取一小段（k 个点），不取到目标**：每轮只需前进一步，确认紧挨的下一段
即可；更远的路相机看不到、手臂也够不着，纳入 B 没意义、还浪费算力。这就是 B 作为
**滑动窗口**（§4.2）的体现。

```python
B = set()
for i in range(reach_idx, min(reach_idx + k, len(P_star) - 1)):
    swept = swept_volume(P_star[i], P_star[i+1], robot_model)
    for v in swept:
        if voxmap[v] == UNKNOWN:        # 只收"仍未确认"的
            B.add(v)
```

##### 该步的其它要点

- **关键澄清**：B 只取「最近的下一段」，不是整条剩余路（§4.2 滑动窗口）。
- **典型情形**：早期 `reach_pt == 当前 config`（前方紧贴着就是未知），B 在鼻子尖前面。
- **坑**：P\* 在真值上无碰撞，挡你的只可能是 UNKNOWN；若扫掠体积碰到 OCCUPIED，
  多半是体素化离散误差，要留安全裕度。

#### ③ 生成候选视点（自由区内可达 + 相机能看向 B）

**做什么**：造出一批既走得到、又看得见 B 的候选构型。回答「相机具体摆哪、朝哪」。

##### (1) 对 B 聚类，取簇心当「要看的目标点」

**为什么聚类**：B 可能是成百上千个零散小格子，不能「每格生成一个视点」；且 B
有时分成几片。

**怎么做**：把 B 按空间位置聚类（k-means / DBSCAN / 网格归并），每个簇取**质心**
（簇内体素坐标平均）当一个**代表点 T**（工作空间 3D 点）。

```
B(一堆未知格子)        聚类           要看的目标点
░░░░░                              
░░░░░   ──────────►       ●T1
   ░░░░                   ●T2   （如果 B 分两片）
```

- 输入：B；输出：1~3 个目标点 `T`。
- 作用：把「看清这片未知云」简化成「看向这几个代表点」。

##### (2) 每个 T 怎么取「若干相机位姿」

一个相机位姿 = **相机眼睛位置 p + 朝向**（6 DOF）。要求：站在确认自由区、朝向 T、
T 落在量程与 FOV 内。

**「具体怎么取若干」= 扫两个维度的组合：**

> **（几档站位距离 d）×（围绕 T 的若干观察方向）= 若干候选相机位姿**

步骤：

1. **选站位距离 d**：相机离 T 要在量程内才拍得清，取几档如 `d ∈ {0.25,0.35,0.45} m`。
2. **撒观察方向**：在 T 周围**半球面**采若干方向（只采「自由区那一侧」，别把相机
   放进障碍）。如均匀球面采样取 8 个方向。
3. **算相机眼睛位置**：`p = T + d · 方向`。
4. **过滤 p**：① `p` 必须在**已确认 FREE** 体素里；② 视线 `p→T` 不被已知障碍挡。
5. **定朝向**：光轴 = `(T−p)/|T−p|`；滚转自由度固定（如相机「上」对齐世界上方）。
6. **FOV 检查**：T（及那一簇）落在视场锥内。

每个通过的 `(p, 朝向)` = 一个候选相机位姿；几档距离 × 若干方向 → 「若干位姿」。

```
                T  (要看的目标点, 在未知区边缘)
              ↗ ↑ ↖
           p3  p2  p1     ← 距离 d、不同方向放相机, 都朝向 T
          (每个 p 必须在 FREE 区; 每个 p+朝向 = 一个候选相机位姿)
       ░░░ 未知区在 T 后面, 相机从自由区这侧看过去
```

```python
def standoff_poses_looking_at(T, voxmap, cam):
    poses = []
    for d in [0.25, 0.35, 0.45]:                       # 几档站位距离(量程内)
        for dir in sample_hemisphere_dirs(n=8, free_side(T, voxmap)):  # 自由侧若干方向
            p = T + d * dir                             # 相机眼睛位置
            if voxmap[voxel_of(p)] != FREE:             # 必须站在确认自由区
                continue
            if sight_blocked_by_known_obstacle(p, T, voxmap):  # 视线别被已知障碍挡
                continue
            look = normalize(T - p)                     # 光轴朝向 T
            pose = make_camera_pose(p, look, up=world_up)
            if target_in_fov(T, pose, cam):             # T 落在 FOV 内
                poses.append(pose)
    return poses
```

##### (3) 眼在手提醒：几何位姿 ≠ 一定到得了

眼在手时，「相机位姿」靠**移动整条手臂、让腕部相机到达 `(p, 朝向)`** 实现。所以：

- 对每个相机位姿做 **cuRobo IK**，求对应关节构型（碰撞感知）；
- 再做**保守可达性过滤**：该构型能否从当前位置**经自由区**走到，到不了的丢弃。

> 本步只定「几何上该把相机摆哪」，**能不能真摆到，由 IK + 可达性过滤决定**。
> **眼在手核心难点**：相机在手腕上，要看向 B 就得让手腕朝向 B，而整条手臂移过去
> 的路径还得全程留在自由区，所以可达视点常常很有限。
> **坑 / 接 stuck**：若没有任何可达视点能看见 B（遮挡/够不着）→ 转「就近揭示」兜底
> （§6）。

---

#### ④ 用真值精确算每个候选揭开 B 多少，选最优

**做什么**：给每个候选打分，挑最优。

- **特权 raycast（仅用于打分）**：从候选相机位姿向真值场景发射光线，取「将被确定」
  体素里当前 UNKNOWN 且 ∈ B 的数量 = `gain`。
- **打分**：`score = gain − λ · path_cost(config → 候选)`，可加「沿 P\* 前进多少」鼓励推进。
- **选择**：`argmax(score)`，GPU 并行算。
- **关键澄清**：这里 raycast 是**假设性**的（如果去那会看到什么），用于比较候选，
  **不改地图**；真正改地图的是 ⑥。

---

#### ⑤ 移动到该视点（前进 + 看清，记录进 GT）

**做什么**：真实地把机械臂移到选中视点，这一步既往目标推进一段、又让相机对准 B。

- **调用**：cuRobo 规划 `config → 选中视点` 的保守无碰撞轨迹（③已验证可达），执行。
- **记录**：这段轨迹接进 GT。
- **关键澄清——途中持续观测**：手臂移动过程中相机一直在拍，沿途就可更新地图，不必
  等到终点才看。更真实、也揭得更多。
- **更新**：`config = 选中视点`。
- **这就是「前进引擎」**：①只是终点判定，真正让机械臂朝目标爬的是每轮的 ⑤。

---

#### ⑥ 观测更新三态图（B 被确认，走廊延长）

**做什么**：在新位置（及沿途）用真实传感器模型观测，把结果提交进持久地图。

- **操作**：从当前相机位姿对真值 raycast → 穿过的体素标 FREE、击中点标 OCCUPIED，
  其余仍 UNKNOWN。
- **关键澄清——和 ④ 的 raycast 区别**：④是「假设性、只为打分、不落地」；⑥是「实际
  发生、提交进 voxmap、机器人真正获得的观测」。两者用同一传感器模型，角色不同。
- **效果**：B 从 UNKNOWN 变 FREE/OCCUPIED，已确认走廊往前延长一段。
- **善后**：`sync_collision_world`（地图变了）；可重算 P\*（自由区扩大后 P\* 可能微调）。

---

#### ⑦ 回到 ①

走廊延长了一段，下一轮 ① 再试规划到目标。剩余未确认走廊越来越短，直到某轮全部
确认、① 成功收尾。

## 5. 简化版 fallback（v1 先跑通）

若第一版不想算 P\* 和阻塞段，可用更糙但稳的目标导向打分：

> `gain = Σ(揭开的未知体素，按「离 start→goal 直线的距离」反比加权)`

即**越靠近「起点→目标直线走廊」的揭示越值钱**。不需全知路径，便宜、鲁棒，但不如
oracle-path 版高效、绕障时不够聪明。

**建议：v1 用加权版跑通，v2 升级到 oracle-path 版产正式 GT。**

---

## 6. 边界情况与失败处理

| 情况 | 处理 |
|------|------|
| B 被遮挡，所有可达视点都看不到 B | **就近揭示**：选能揭开最多「任意未知」的可达视点（或 `look_around` 小幅摆动），扩大自由区 → 新视点变可达 → 重试 |
| 连续 K 轮 reach_pt 没前进、自由区没扩大 | 判定**卡死**，标记该场景 |
| 真值上 P\* 不存在 | 场景**真不可行**，标记并跳过 |
| B 为空、但仍规划不到目标 | 走廊其实已确认，是规划器没找到 → **加大 cuRobo 种子 / 尝试数** |
| P\* 随移动而变 | 每次迭代**重算 P\***（算力足，无妨） |

---

## 7. 「特权」的正当性与一个注意点

**正当性**：记录的 GT 轨迹**实际移动仍只走已确认自由区**，完全遵守保守约束；真值
**只用来决定「往哪看」**，让探索高效。学生模型学的是「真实观测 → 动作」，推理时
不需要真值。**老师靠特权产高质量标签，是 teacher-student 的标准做法，不是有害作弊。**

**注意点（务必想清楚）**：若模型将来**只靠部分观测**就要复现这种探索行为，则 GT
行为最好**在部分观测下也「说得通」**。oracle-path NBV 天然满足——「朝要去的方向看
一眼」是从部分观测也能学会、合理的策略。但若让特权专家做出「凭空知道该看某个
观测上毫无线索的犄角旮旯」这类动作，模型将学不会。

> **oracle-path 版本是安全的；纯靠真值跳跃式选点则有风险。**

## 8. 详细伪代码（顶层）

```python
def best_next_view_using_oracle(motion_gen, config, voxmap, truth_scene, goal_config):
    # 1) 真值上的"全知最优路径"
    P_star = plan_on_truth(config, goal_config, truth_scene)   # cuRobo vs 真值场景
    if P_star is None:
        return None, "scene_infeasible"

    # 2) 沿 P* 逐点向前扫求 reach_pt，再取前方一小段的 UNKNOWN 体素 = B
    reach_idx = 0
    for i in range(len(P_star) - 1):
        swept = swept_volume(P_star[i], P_star[i+1], robot_model)
        if all(voxmap[v] == FREE for v in swept):
            reach_idx = i + 1
        else:
            break
    B = set()
    for i in range(reach_idx, min(reach_idx + K_LOOKAHEAD, len(P_star) - 1)):
        swept = swept_volume(P_star[i], P_star[i+1], robot_model)
        B |= {v for v in swept if voxmap[v] == UNKNOWN}
    if not B:
        return None, "corridor_confirmed"     # 走廊已确认,应能直接规划到目标

    # 3) 朝 B 生成"确认自由区内可达"的候选视点
    cands = []
    for t in cluster_centroids(B):
        for cam_pose in standoff_poses_looking_at(t, voxmap, cam):  # 站位取在 FREE 区
            cfg = ik(motion_gen, cam_pose)
            if cfg is not None and motion_stays_in_free(config, cfg, voxmap):
                cands.append(cfg)

    # 4) 真值精确算"揭开 B 多少",减路径代价
    best, best_score = None, -1e9
    for cfg in cands:
        revealed = raycast_reveal(camera_pose(cfg), truth_scene)   # 特权 raycast
        gain = len(revealed & B)                  # 只数揭开阻塞段的部分
        score = gain - LAMBDA * path_cost(config, cfg)
        if score > best_score:
            best, best_score = cfg, score

    if best is None:
        return greedy_reveal_nearest_unknown(config, voxmap, truth_scene), "fallback"
    return best, "ok"
```

---

## 9. 小结

> **特权 NBV = 在真值上先算出「全知最优路径 P\*」（一串关节构型），沿它逐点向前
> 扫出能保守走到的尽头 reach_pt，取其前方一小段里仍 UNKNOWN 的体素为阻塞段 B；
> 再对 B 聚类成几个目标点，在自由区内『几档距离 × 若干方向』生成朝向它们的候选
> 相机位姿，经 IK 与可达性过滤后，用真值精确算各候选能揭开 B 多少，挑最多的。**
>
> 本质是「**朝你要走的理想路径往前看一段、确认、再走**」；单轮只挑下一个视点，
> 完整的初始→目标过程是主循环把它重复 N 轮、逐段推进（§4.5）。真值只用于选对看
> 的方向、不用于放宽移动约束——既高效又合规。
