# 焊缝在线主动巡检 —— 两阶段混合管线

本文档为以下具体场景设计算法管线：

- **固定底座** 6-DoF 机械臂（你的 UR12e）
- 工件上一条 **连续焊缝**（约 20–50 个采样点），每点都需要被相机至少看到一次
- 观测距离硬约束：**30–60 cm**
- 焊缝在臂可达空间内，但工作空间内有多个障碍物
- 初始位姿下相机看不到焊缝，必须**逐步探索**才能找到观测路径

它不是单纯探索（DSVP/GBPlanner2/TARE），也不是单点目标 NBV（WG-NBVP），而是 **"在线主动巡检 (online active inspection)"** —— 探索和覆盖交织。

---

## 一、为什么没有现成单一算法直接吻合

| 候选 | 缺什么 | 但什么部分可借用 |
|------|--------|------------------|
| DSVP | 单点 NBV 贪心，不擅长连续覆盖 | RRT* 局部扩张 |
| GBPlanner2 | 纯探索，无目标先验 | 全局图 + dead-end 回退 |
| TARE | 纯覆盖，假设地图已知 | OP/TSP 多 viewpoint 路线求解 |
| WG-NBVP | ROI 是体素，不是曲线；单 NBV，不出 tour | 多目标加权评分 |

**结论**：没单赢家。最划算的方案是把 GBPlanner2（探索阶段）和 TARE（覆盖阶段）的核心拼起来。

## 二、问题数学化

### 2.1 输入

```text
- q_0:           机械臂初始关节构型 (6-D)
- M_0 = ∅:       初始 OctoMap, 全 unknown
- {P_1,...,P_N}: 焊缝离散采样点, 每个 P_i ∈ ℝ³ 加切线方向 t_i
- d_min, d_max:  观测距离区间, 0.3m, 0.6m
- arm_model:     URDF / cuRobo 配置
- workpiece_USD: 工件 USD (用于碰撞检测和 raycast 真值)
```

### 2.2 输出

```text
一段连续可执行的关节空间轨迹, 满足:
  1. 整条轨迹无碰撞 (整臂 vs 当前 OctoMap)
  2. 沿轨迹某些 keyframe 处, 相机能"合格观测"焊缝点 P_i
  3. 所有 P_i 都被覆盖 (S = {1,...,N})
```

### 2.3 "合格观测" P_i 的硬约束

```text
viewpoint v = (camera_pos, camera_dir) 合格观测 P_i 需要:
  ① 距离: d_min ≤ ‖camera_pos − P_i‖ ≤ d_max
  ② 在视锥内: P_i 在 v 的相机视锥范围内
  ③ 视线无障碍: raycast(camera_pos, P_i) 全程已知 free
  ④ 入射角合理: 视线方向与焊缝切线 t_i 夹角 ∈ [30°, 90°]
                (避免视线和焊缝平行)
```

四个条件全满足，记 P_i ∈ S（已观测集）。

## 三、整体两阶段流程

```text
                    ┌─────────────────────────────┐
                    │   START: q_0, M_0 = ∅       │
                    └──────────────┬──────────────┘
                                   │
                                   ▼
          ┌───────────────────────────────────────────┐
          │  Phase 1: Exploration (GBPlanner2-style)   │
          │  目标: 让相机第一次"合格观测"任一 P_i        │
          │                                             │
          │  ┌─ 维护全局图 G(V,E)                       │
          │  ├─ 每 cycle 在当前位置周围长 box 扩张图    │
          │  ├─ 评分 = volumetric_gain                  │
          │  ├─ Dijkstra 选最优, 执行第一段              │
          │  └─ 死胡同 → Global frontier 跳转           │
          │                                             │
          └──────────────┬─────────────────────────────┘
                         │ 任一 P_i 合格观测条件首次满足
                         ▼
          ┌───────────────────────────────────────────┐
          │  Phase 2: Coverage (TARE-style with       │
          │            distance constraint)            │
          │  目标: 把所有 P_i 装进 S                    │
          │                                             │
          │  ┌─ 在每个未观测 P_i 周围球壳撒 viewpoint  │
          │  ├─ 过滤: IK + 碰撞 + 视线 + 入射角        │
          │  ├─ OP/TSP 求最优 tour (路径预算 L_max)    │
          │  ├─ 执行前段, 更新 M 和 S                  │
          │  └─ 卡住 → fallback 回 Phase 1             │
          │                                             │
          └──────────────┬─────────────────────────────┘
                         │ S = {1,...,N}
                         ▼
                    ┌─────────────┐
                    │   DONE      │
                    └─────────────┘
```

允许 Phase 2 → Phase 1 的回退（虚线，下面详述）。

## 四、Phase 1：Exploration 详细步骤

### 4.1 数据结构

```text
G = (V, E)            # 全局图, 累积不重置
   V: 节点 = 关节构型 θ ∈ ℝ⁶ (或末端 SE(3))
   E: 边 = 已验证可走的直线段
M: OctoMap            # 全局占据栅格
S: set[int] = {}      # 已观测 P_i 索引集合
```

### 4.2 每 cycle 步骤

详见 [GBPlanner2.md 第五章](GBPlanner2.md)。这里只列出**针对你场景的关键改动**：

#### 改动 1：节点是构型，不是 3D 点

```text
原版: V_local = 3D 立方体, 节点 = 3D 位置
你版: V_local = 关节空间的小立方体 (每关节 ±0.3 rad)
      或末端笛卡尔小盒 + IK
```

#### 改动 2：碰撞检测用 cuRobo

```text
edge (θ_a, θ_b) 是否可走:
   for k in 0..K:
     θ = lerp(θ_a, θ_b, k/K)
     if cuRobo.check_collision(θ, octomap_voxels):
        return False
   return True
```

#### 改动 3：volumetric_gain 加目标偏置

```text
gain(v) = sum(unknown 体素 in 视锥, 用 raycast)
        + α · max(0, cos(angle(camera_dir, P_weld - v.camera_pos)))
```

第二项让相机优先朝焊缝大致方向看，加速进入 Phase 2。

#### 改动 4：每 cycle 检查 Phase 切换条件

```text
for i in 1..N:
    if not i in S:
        if 满足合格观测条件 (距离/视锥/视线/入射角):
            S.add(i)
            
if len(S) >= 1:
    return PHASE_2_TRIGGER     # 切到 Phase 2
```

只要**第一个**焊缝点被合格观测，立即切。

#### 改动 5：Global 阶段（dead-end 回退）

完全沿用 GBPlanner2 的 Global step：在全局图上找仍有 gain 的旧节点，Dijkstra 走过去。

## 五、Phase 2：Coverage 详细步骤

### 5.1 候选 viewpoint 生成

对每个未观测 P_i（即 i ∉ S）：

```text
1. 确定球壳采样区:
   center = P_i
   inner_radius = 0.3 m
   outer_radius = 0.6 m

2. 在球壳上撒 K_view 个候选位置 (论文经验值 K_view = 20):
   - 均匀球面采样 + 半径在 [0.3, 0.6] 抖动
   - 排除朝向焊缝切线方向的位置 (入射角约束)

3. 每个候选位置, 朝向锁定为指向 P_i:
   camera_dir = (P_i - camera_pos) / ‖...‖
   
4. 验证 IK + 碰撞 + 视线 + 入射角 (合格观测的 4 条):
   合格 → v_ij 加入候选池
   失败 → 丢弃

5. 标记每个候选 v_ij 还能"顺路覆盖"哪些其他 P_k:
   for k in 1..N, k ∉ S:
     if v_ij 也能合格观测 P_k:
        v_ij.bonus_covered.add(k)
```

```text
俯视球壳采样可视化:

           o   o   o
        o    \ | /    o
       o   --- P_i ---  o     o = 候选 viewpoint
        o    / | \    o          线 = 各个朝向都指向 P_i
           o   o   o
       
       0.3m ─┤       ├─ 0.6m
```

### 5.2 距离矩阵（cuRobo trajectory cost）

对所有候选 viewpoint 两两计算"机械臂从 v_a 到 v_b 的真实关节空间轨迹长度"：

```text
dist[a][b] = cuRobo.compute_trajectory(v_a.theta, v_b.theta).length

注意:
  - 用关节空间轨迹长度, 不用末端笛卡尔距离 (后者不准)
  - 用 trajectory cost 而非 path planning, 节省时间
  - 对称: dist[a][b] = dist[b][a] (假设)
```

如果候选 ≤ 50 个，全矩阵 ~1250 次 cuRobo 调用，每次 ~10 ms → 总 ~12 s 可接受。
如果候选 > 50，用两层过滤：先笛卡尔距离粗筛保留 top 50，再算精确轨迹。

### 5.3 求解 Orienteering Problem (LKH-3)

```text
节点: 当前位置 q_now ∪ 所有候选 viewpoint
起点: q_now
路径预算: L_max = 1.0 (关节空间总移动量, rad·关节数)
节点 reward:
  reward(v_ij) = (1 if i ∉ S else 0)
              + sum(1 for k in v_ij.bonus_covered)
              + β · (顺路扫到的 unknown 体素数)

求解:
  tour = LKH3.solve_OP(
      start=q_now,
      nodes=候选池,
      distance=dist,
      rewards=reward,
      budget=L_max
  )
```

返回 tour = 一个有序 viewpoint 序列。

### 5.4 路径增强 + 平滑

```text
- 沿 tour 在每两个相邻 viewpoint 之间插入 5-10 个时间均匀的中间构型
- 用 cuRobo 的 trajopt 或 spline smooth 整条轨迹
- 避免末端急转弯, 加速度连续
```

### 5.5 执行 tour 前段（滚动）

```text
执行 tour 的前 STEP_LENGTH 长度 (~0.2-0.3 m 末端位移)
执行过程中相机持续输出 RGB-D, 持续更新 M
执行结束:
  for i in 1..N:
     if 检测到 i 满足合格观测条件:
        S.add(i)
重新进入 Phase 2 一个 cycle (重算 OP), 直到 S = {1,...,N}
```

### 5.6 卡住检测 → fallback 到 Phase 1

```text
if 连续 K_STUCK cycles 都没有候选 viewpoint 通过 4 条合格观测验证:
  # 说明地图里"已知 free"区域不足以让候选过视线检查
  goto Phase 1     # 继续扩张地图
```

## 六、Phase 1 ↔ Phase 2 切换逻辑

```text
                      ┌────────────┐
                      │  Phase 1   │
                      └──┬──────┬──┘
                         │      │
            首次 P_i ∈ S │      │ 已观测目标全失效
                         │      │ (比如焊缝点改了)
                         ▼      │
                      ┌────────────┐
                      │  Phase 2   │
                      └──┬──────┬──┘
                         │      │
            S = {全部}   │      │ 卡住 K_STUCK cycles
                         │      │
                         ▼      ▼
                      DONE    回 Phase 1
```

**两个方向都允许**，避免某一阶段陷死。

## 七、超精简伪代码

```python
def online_active_inspection(q_0, seam_points, octomap, P_weld_center):
    M = octomap
    S = set()                    # 已观测 P_i 索引
    G = Graph()                   # 全局图 (Phase 1 用)
    G.add_node(q_0)
    q = q_0
    phase = 1
    stuck_count = 0
    
    while len(S) < len(seam_points):
        if phase == 1:
            # ===== Phase 1: Exploration =====
            sub_result = gbplanner2_one_cycle(q, G, M, P_weld_center)
            q = sub_result.next_q
            execute_arm(q)
            update_map(M)
            
            # 切换检测
            for i, P_i in enumerate(seam_points):
                if i not in S and is_valid_observation(q, P_i, M, seam_points):
                    S.add(i)
            
            if len(S) >= 1:
                phase = 2
                stuck_count = 0
        
        elif phase == 2:
            # ===== Phase 2: Coverage =====
            candidates = []
            for i, P_i in enumerate(seam_points):
                if i in S: continue
                # 球壳采样
                positions = sample_spherical_shell(P_i, r_in=0.3, r_out=0.6, K=20)
                for pos in positions:
                    cam_dir = normalize(P_i - pos)
                    v = Viewpoint(pos, cam_dir, target=i)
                    if not pass_observation_filters(v, P_i, seam_points, M):
                        continue
                    theta = inverse_kinematics(v)
                    if theta is None or arm_collides(theta, M):
                        continue
                    v.theta = theta
                    # 顺路覆盖
                    for k in range(len(seam_points)):
                        if k != i and k not in S:
                            if is_valid_observation(v, seam_points[k], M, seam_points):
                                v.bonus_covered.add(k)
                    candidates.append(v)
            
            if not candidates:
                stuck_count += 1
                if stuck_count >= K_STUCK:
                    phase = 1     # fallback
                continue
            
            # 距离矩阵
            dist = compute_distance_matrix(q, candidates)
            
            # OP 求解
            rewards = [1 + len(v.bonus_covered) for v in candidates]
            tour = LKH3.solve_OP(start_idx=0,
                                 dist=dist,
                                 rewards=rewards,
                                 budget=L_MAX)
            
            # 路径增强 + 执行前段
            full_traj = curobo.trajopt(tour)
            execute_arm_segment(full_traj, length=STEP_LENGTH)
            update_map(M)
            
            # 更新 S
            for i, P_i in enumerate(seam_points):
                if i not in S and is_valid_observation(q, P_i, M, seam_points):
                    S.add(i)
            
            stuck_count = 0
    
    return final_traj


def is_valid_observation(viewpoint, P_i, M, seam_points):
    """合格观测的 4 条硬约束"""
    pos, cam_dir = viewpoint.pos, viewpoint.dir
    # ① 距离
    d = norm(pos - P_i)
    if not (0.3 <= d <= 0.6): return False
    # ② 视锥内
    if not in_camera_frustum(P_i, pos, cam_dir): return False
    # ③ 视线无障碍
    if raycast_blocked(pos, P_i, M): return False
    # ④ 入射角
    tangent = seam_points[i].tangent
    angle = acos(abs(dot(cam_dir, tangent)))
    if not (radians(30) <= angle <= radians(90)): return False
    return True
```

## 八、完整例子（用你 pkl 的真实数据）

数据：

- 末端起点 q_0 → camera_pos ≈ (24.4, 0.5, 9.8)
- 焊缝 20 个点，沿 y 轴方向，从 (24.025, 0.157, 11.432) 到 (24.035, 0.157, 11.432)
- P_weld 中点 (24.030, 0.157, 11.432)
- 距离差 ~1.85 m，焊缝在初始视锥外

### Phase 1: Cycle 1–N（约 N=10 cycles）

```text
Cycle 1:
  - 在 q_0 周围长全局图, 撒 100 个候选 6-D 构型
  - 每个候选 cuRobo IK + 碰撞通过的留下
  - volumetric_gain 算未观测体素数
  - 朝 P_weld 方向偏置项让相机优先朝上方
  - Dijkstra 选最优, 移动 0.1m

Cycle 2-9:
  - 末端逐渐移向 P_weld 方向
  - 每 cycle 检查焊缝点合格观测, 都不满足 (太远, ~1.5m)
  - 图持续累积

Cycle 10:
  - 末端到达 (24.1, 0.3, 10.8) 附近
  - P_5 (焊缝某点) 满足全部 4 条合格观测
  - S = {5}
  - 切到 Phase 2
```

### Phase 2: Cycle 11–M（约 M=15 cycles）

```text
Cycle 11:
  - 19 个未观测点, 每点撒 20 个候选 = 380 候选
  - 过滤后剩 ~80 个合格 viewpoint
  - 距离矩阵 80×80 (cuRobo)
  - LKH-3 解 OP: tour = [v_for_P3, v_for_P7, v_for_P12, ...]
  - tour 第一段移动 0.25m
  - 执行后 S = {3, 5, 7}

Cycle 12-25:
  - 每 cycle 重算 OP (因为 S 增长, candidate pool 减少)
  - tour 的"顺路 bonus"机制让一次移动覆盖多点
  - 偶尔遇到立柱挡住某段, 候选清空 → 卡住计数 +1

Cycle 20 (假设):
  - 焊缝末端被立柱挡, 连续 K_STUCK=3 次没有可行候选
  - fallback 回 Phase 1, 沿全局图绕到立柱另一侧
  - 在新位置重新触发 Phase 2

Cycle 25:
  - S = {1,...,20}, 任务完成
```

## 九、超参清单

| 超参 | 含义 | 推荐值 | 调大效果 | 调小效果 |
|------|------|--------|----------|----------|
| `K_view` | 每 P_i 球壳采样数 | 20 | 候选多, OP 慢 | 候选少, 可能漏好点 |
| `r_in, r_out` | 球壳内外半径 | 0.3 / 0.6 m | 候选区域大, 灵活 | 拒绝率高, 解空间小 |
| `L_max` | OP 路径预算 (关节空间) | 1.0 rad·n | tour 长, 一次扫多点 | tour 短, 频繁重规划 |
| `STEP_LENGTH` | 每 cycle 执行多长 | 0.2-0.3 m 末端 | 周期少, 地图过期 | 周期多, 计算慢 |
| `K_STUCK` | fallback 触发耐心 | 3 | 不容易回 Phase 1 | 频繁切换 |
| Phase 1 `bounding_box` | 局部图增长盒 | 0.3 m | 节点广 | 局部细 |
| Phase 1 `α` | 目标方向偏置权重 | 0.3-1.0 | 直奔目标 | 自由探索 |
| Phase 2 `β` | 顺路 unknown bonus | 0.05-0.2 | 顺手扫地图 | 纯覆盖 |
| 入射角范围 | 焊缝法线允许的夹角 | [30°, 90°] | 候选多, 质量参差 | 候选严, 拒绝率高 |
